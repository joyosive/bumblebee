Phase 1 — Campaign Submission
NGO

Posts Campaign Request

NGO messages the Telegram bot with their campaign details — what they do, where, how much they need. Can chat naturally or use /campaign command.

↓
FacilitatorBee

Receives & Stores Campaign

Extracts title, description, sector, country, and funding goal from the conversation. Saves to database. Notifies EvaluatorBee.

Phase 2 — Evaluation
EvaluatorBee

Reviews & Scores Campaign

Evaluates viability: sector alignment, goal clarity, funding reasonability. Scores the campaign.

↓
EvaluatorBee

Sets 3 Milestones

Breaks the campaign into 3 milestones with clear deliverables. Sends fund estimate back to NGO.

NGO

Receives Milestone Plan

"Your campaign is approved. 3 milestones set: M1 (1 XRP), M2 (1 XRP), M3 (1 XRP). Waiting for funding."

Phase 3 — Auto Fund Allocation
Impact Investor (Pre-funded)

Treasury Already Loaded

The impact investor has already deposited XRP into TreasuryBee's wallet. This is a one-time setup — like loading an escrow treasury. No browsing, no picking, no delays.

↓
TreasuryBee

Auto-Allocates Funds

Once EvaluatorBee approves a campaign, TreasuryBee checks pool balance and automatically allocates funds. No human decision needed — the Bees handle it.

↓
TreasuryBee

Creates 3 Milestone Escrows

Locks allocated funds into 3 separate escrows to the NGO's wallet (33% / 33% / 34%). Each has its own crypto-condition lock. Funds release only when the matching milestone is verified.

XRPL Escrow Create ×3
Phase 4 — Milestone Execution
NGO

Works on Milestone & Submits Evidence

NGO completes the work, then sends photos, invoices, receipts, or reports via Telegram using /submit.

↓
VerifierBee

Reviews Evidence

Checks submitted evidence. If good → approves. If not → sends specific feedback for resubmission.

XRPL Credential
↓
TreasuryBee

Releases Milestone Payment

Unlocks the escrow for this milestone. XRP goes directly to the NGO's wallet. On-chain, verified, transparent.

XRPL Escrow Finish
or
NO EVIDENCE SUBMITTED

Campaign Stalled — Funds Reclaimed

If the NGO fails to submit evidence within the deadline, the milestone expires. TreasuryBee cancels remaining escrows and returns funds to the pool. No further grants allocated for this campaign.

XRPL Escrow Cancel
Phase 5 — Completion & Trust Score
ReviewerBee

Calculates Trust Score

Scores the NGO (0-100) based on: milestone completion speed, evidence quality, and how well funds were used.

↓
ReviewerBee

Publishes On-Chain

Trust score recorded permanently on XRPL via Oracle. Completion credential issued to NGO. Fully auditable.

XRPL Oracle + Credential
NGO

Campaign Closed

"All done. Trust score: 85/100. Published on-chain. Your completion credential is issued."

Wallet Architecture — 6 Wallets

TreasuryBee Wallet

Impact investor's treasury. Pre-funded once. Auto-allocates to approved campaigns.
Pre-funded via faucet (max 10 XRP active pool)
The investor IS this wallet. No separate donor wallet. Needs 16 XRP reserve when 3 escrows active. Reserve returned when escrows close.

EvaluatorBee Wallet

DID registration, signs campaign evaluations
Funded by testnet faucet (~100 XRP)
Minimal XRP usage. Just needs base reserve for identity.

FacilitatorBee Wallet

DID registration, signs milestone definitions
Funded by testnet faucet (~100 XRP)
Minimal XRP usage. Identity + signing only.

VerifierBee Wallet

DID registration, issues milestone credentials
Funded by testnet faucet (~100 XRP)
Credential transactions cost ~0.00001 XRP each.

ReviewerBee Wallet

DID registration, oracle publish, completion certs
Funded by testnet faucet (~100 XRP)
Oracle + credential transactions. Minimal cost.

Test NGO Wallet

Receives milestone payments
Funded by testnet faucet (~100 XRP)
Receives escrow releases. 1 XRP per milestone in test scenario.

On-Chain Identity (DID)

Every action a Bee takes on XRPL is signed by its Decentralized Identifier. Anyone can verify which Bee did what.
